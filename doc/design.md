# MetaEditor 设计文档

MetaEditor 是一个基于 MoonBit 的编辑器开发框架（元编辑器）。它提供一套面向编辑器场景的底层运行时与工程基础设施，使开发者能够高效构建具有复杂交互、可持续演进、适合 AI 协作的软件编辑器系统。

## 1. 项目目标

- **响应式架构**：提供专为编辑器场景优化的响应式 UI 运行时。
- **状态管理**：内置自动保存、撤销重做（Undo/Redo）、快照恢复等核心能力。
- **AI 协作**：原生支持 AI 代理接入，提供结构化命令接口（CLI）与图形界面（Web UI）双重交互方式。
- **多端复用**：使编辑器核心逻辑可跨 Web、命令行及远程环境复用。

## 2. 系统架构

系统采用分层架构设计，确保核心逻辑与表现层解耦：

### 2.1 编辑器运行时层 (Runtime Layer)
- **响应式系统**：实现细粒度更新机制。
- **组件模型**：管理 UI 节点的生命周期与资源清理。

### 2.2 编辑器状态层 (State Layer)
- **文档状态 (Document State)**：编辑器处理的核心数据模型。
- **界面状态 (UI State)**：临时性的视图状态（如滚动位置、选中范围）。
- **历史管理**：操作式状态变更，支持撤销重做与自动保存。
- **快照机制**：支持快速导出与恢复编辑器完整状态。

### 2.3 编辑能力层 (Capability Layer)
- **编辑模型**：定义具体的编辑行为（如文本输入、多选、搜索）。
- **交互适配**：为 Web UI 和 CLI 提供统一的状态操作接口。
- **AI 接口**：面向 AI 代理的自动化操作支持。

## 3. 核心模块划分

- `ui`：面向编辑器的响应式 UI 运行时。
- `state`：操作式状态管理、历史记录与快照恢复。
- `cli`：面向 AI 或自动化脚本的命令行入口。
- `text-editor`：文本编辑领域的核心逻辑实现。
- `parser`：面向编辑器场景的解析框架（支持增量更新与错误恢复）。

## 4. 关键特性设计

### 4.1 响应式 UI 运行时
针对编辑器高频交互特点，采用类似 Signal 的细粒度响应式模型，避免全局重新渲染，提升性能。

### 4.2 状态快照与持久化
状态系统支持序列化为快照。这不仅用于自动保存，还允许 AI 代理在断点后直接恢复环境继续工作。

### 4.3 AI 与人类协同
- **图形界面 (Web)**：服务于人类用户的直观操作。
- **命令行接口 (CLI)**：服务于 AI 代理的结构化读写操作。
两者共享同一套编辑器状态模型。

## 5. 技术选型

- **开发语言**：MoonBit (面向 Wasm 的高性能语言)。
- **目标平台**：Web / Wasm。
- **交互方式**：Web UI + CLI。
- **测试体系**：MoonBit 自带单元测试 + 关键链路集成测试。

## 6. 风险与规避

- **动态特性迁移**：MoonBit 是静态类型语言，需要将原 JavaScript 版本中的动态代理（Proxy）等特性重新设计为静态类型友好的响应式模型。
- **性能优化**：解析器在 MoonBit 下的执行效率需要通过实验进一步验证。
- **UI 验证**：通过可复现的示例场景和状态层深度测试来弥补复杂交互自动验证的不足。
